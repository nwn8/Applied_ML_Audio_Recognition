# Project Instructions


###  Reproduction and local setup

1.  Clone the repository at [https://github.com/nwn8/Applied_ML_Audio_Recognition]

2.  Follow the setup process in requirements.txt to create a virtual environment and install packages

* note that some packages like openai-whisper and chromaprint require extra steps on your device outlined in requirements.txt

### Creating Source Data

3. Add 25 or more MP3 audio files to the "songs" folder.  This will be the foundation of the source data.

4.  run "py create_database.py" in the terminal  ---  This creates the database for storing the data with the following schema

Songs
-------
song_id (pk) auto increment
title text
duration num
lyrics text
cleaned_lyrics text
word_count num
unique_word_count num
word_freq object

Fingerprints
-------
fingerprint_id  (pk) Auto Increment
song_id (Fk)
fingerprint (generated by chromaprint) text


Audio_Features  MFCC- a compact numeric summary of what a song sounds like  (Mel-Frequency Cepstral Coefficients)
---------
feature_id (pk) auto increment
song_id (fk)
tempo num
mfcc_1 num
... 
mvcc_20

5.  Run "py loader.py" in the terminal  -  This will extract all of the audio data from the files in the "songs" folder and add the data to the songs_database.db

###  Exploratory Data Analytics

6.  Now that there is data in the database you can explore the datas various attributes and understand the data using Project_EDA.ipynb

# Load data into database 

create a group of 25 - 50 mp3 songs in the songs folder.  

run loader.py.  

This will do 3 things 
1. Lyrics
    Extract lyrics using Open ai whisper.
    create a word frequency count of the lyrics.  
    crate a cleaned version of the lyrics.
    Crate a word count of the lyrics.
    Create a unique word count of the lyrics

2. Fingerprint
   Use Chromaprint in AcoustID/Chromaprint to create a unique song fingerprint 

3. Mel-Frequency Cepstral Coefficients (MFCC- a compact numeric sumary of what a song sounds like)
   Use Librosa to create 20 unique charactics of the song

All data gets stored in the database created.
